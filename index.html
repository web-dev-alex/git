git

describe the way git works

install git

1. öffne visual studio code

2. auf der linken seite befindet sich eine vertikale bar, in der das source control icon angewählt wird

3. dann öffnet sich ein neues fenster, welches den direkten link zur installation von git bereit hält

4. auf der externen seite installieren und visual studio neu starten

5. für die folgenden befehle kann das terminal in visual studio code verwendet werden



basisfunktionen in der richtigen Reihenfolge

1. im terminal das verzeichnis der gewünschten datei auswählen

2. dann mit dem Befehl: git init leeres repository initialisieren

3. mit dem Befehl: git status wird eine übersicht von verfolgten/nicht verfolgten dateien gezeigt

4. für das staging von einer datei wird der Befehl: git add [dateiname]

mit dem zuvor genannten Befehl wird die Datei verfolgt(track the file)

mit dem Befehl: git add . können alle dateien in dem gewählten ordner gestagte werden

optional: mit dem Befehl: git status kann nach der Verwendung des git add [dateiname] das erfolgreich gestagte file kontrolliert werden (die datei wird jetzt getracked)

5. die datei/file ist jetzt bereit für ein commit/festlegen

6. vor dem Anlegen eines Speicherpunktes muss ein account angelegt werden

Befehl: git config --global user.name "[username]"

Befehl: git config --global user.email "[email]"

7. um einen neuen Speicherpunkt anlegen zu können wird der Befehl: git commit -m "Startpunkt" verwendet

bei dem genannten Befehl wird mit dem Speicherpunkt eine Message(die nachricht im Beispiel oben ist Startpunkt) hinterlegt

zur Kontrolle kann der Befehl: git log verwendet werden

datei auf den letzten Speicherpunkt zurücksetzen

1. anzeigen des letzten Speicherpunktes und des aktuellen Zustandes der Datei/file mit dem Befehl: git diff [dateiname]

2. löschen des aktuellen Zustandes und zurücksetzen auf den alten Speicherpunkt mit dem Befehl: git checkout [dateiname]
